使用以下命令，可以查看运行时目录：

:set rtp?

以上文件夹是运行时文件夹，vim通常会搜索以下三个目录下的运行时文件夹：

  • $HOME/vimfiles（windows上是你的用户文件夹根目录下的vimfiles ）
  • $VIM/vimfiles ( vim下的vimfiles)
  • $VIMRUNTIME (vimxx，比如vim72)
这样可以在不同的状况加载不同的插件和设置（比如多人共同使用，每个人可以有自己的vim设置） 你可以通过直接修改 rtp 来增加/修改运行时目录,如 :set runtimepath+=c:/blahblah/vimfiles 这样你也可以把插件等文件夹放在一个完全不同的地方。 :help rtp :help dos-locations vimfiles目录中的设置，会覆盖runtime目录中的设置。例如，vimfiles/plugin/myplug.vim 将优先于 $VIMRUNTIME/plugin/myplug.vim 而被载入。

安装目录

使用以下命令，可以查看$VIM所代表的安装目录：

:echo $VIM

对于Linux操作系统，Vim将被安装在以下目录：

/usr/share/vim

对于Windows操作系统，Vim将被安装在以下目录：

C:\Program Files\vim

对于默认安装目录，您不应该修改以上安装目录中的内容，也不应该在其中存放用户相关的文件。主要包括以下内容：

  • vimXX：带上版本号的程序文件夹，可以让不同版本共存。
  • vimfiles 是用户文件夹，你自己装的插件等东西最好放在这里。

用户目录

使用以下命令，可以查看$HOME所代表的用户目录：

:echo $HOME

Unix下的 $HOME/.vim/ 目录 are for any customizations that should not be over-written when upgrading to a new version of vim. The custom configuration directory

commonly located at ~/.vim. This contains all of our personal modifications so we don’t mess with the default configuration. From what I can tell, you can put any directory that matches the names in the install directory, and the files within will behave as if you had dropped them in the installation directory. Again, you can do this to override the default behavior without messing anything up so you can easily take your changes to other machines. For an example of what my ~/.vim directory looks like, check it out.

在首次使用Vim之前，可以使用以下命令来创建vimrc配置文件：

$ touch ~/.vimrc

Linux: /home/username/.vimrc Windows: c:\documents and settings\username\_vimrc Whatever you change in the personal vimrc file will overrule any previous setting made in the global vimrc file.
