:echo $vim vimfiles目录中的设置，会覆盖runtime目录中的设置。 for example, if you have vimfiles/plugin/myplug.vim and $VIMRUNTIME/plugin/myplug.vim, the one in vimfiles is loaded. Directory structure under vimfiles should indeed mimick that under $VIMRUNTIME :help runtimepath The $VIM\vimfilel\ directory (DOS) and the $HOME/.vim/ directory (unix) are for any customizations that should not be over-written when upgrading to a new version of vim. The question of which file overrides which is a little complicated. For example, on my W95 machine, 'runtimepath' includes the directories D:\VIM/vimfiles (a.k.a. $VIM\fimfiles) D:\VIM\vim61 (a.k.a. $VIMRUNTIME) D:\VIM/vimfiles/after in that order, which determines search order. Now, if I use :runtime, then the first file found is used; it overrides the other(s). If I use :runtime!, however, all found files are :source'd, so the last one found overrides the others. (This is why we have the after/ directory!) Some files check for the existence of a variable; if it exists, they bail out. In this case, first file wins again. :help :runtime :help ftplugin-overrule
